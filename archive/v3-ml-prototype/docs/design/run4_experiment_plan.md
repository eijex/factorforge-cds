# Run 4 Experiment Plan: v3-alpha Pseudo-Label Training

> Status: draft, pre-training only  
> Boundary: no full training without explicit approval  
> Production fallback: v2 remains required

## Objective

Run 4 should test whether v3 can learn from validated v2 pseudo-labels and constrained candidates better than native-CDS imitation. It must not claim real protein-yield improvement.

Run 4 should prove:

- v3 can train on expression-oriented pseudo-labels rather than native CDS labels.
- decoded candidates preserve the input amino acid sequence.
- candidates pass hard validator checks before recommendation.
- v3 is compared against v2, not positioned as a replacement.

Run 4 should not prove or claim:

- wet-lab yield improvement;
- regulatory suitability;
- production replacement of v2;
- superiority without ablation and validator-backed benchmark evidence.

## Dataset Strategy

Primary training target:

- validated v2 pseudo-label CDS generated by `scripts/1_data_preparation/generate_v2_pseudolabels.py`;
- JSONL rows must include protein sequence metadata so synonym masks can be built;
- rows with validator failure should be excluded by default.

Reference/evaluation sets:

- native CDS as background/reference only;
- v2 pseudo-labels as teacher baseline;
- constrained-oracle candidates where feasibility analysis permits;
- safe synthetic benchmark proteins from `tests/fixtures/benchmark_proteins.py`;
- random synonymous baseline for sanity checks.

## Loss Function

Initial Run 4 loss:

```text
CE_to_pseudo_label
  - lambda_cai * expected_log_CAI
  + bounded_GC_penalty
```

Bounded GC:

```text
lambda_low * relu(gc_low - expected_gc)
+ lambda_high * relu(expected_gc - gc_high)
```

Initial config:

- `gc_low: 0.40`
- `gc_high: 0.55`
- `gc_weight: 0.2`
- `expected_log_cai_weight: 0.05`

## Synonymous Constraints

Training:

- `CodonDataset` builds `synonym_mask` from the protein sequence.
- expected GC and expected log CAI use the mask.
- rows without protein sequence metadata fail closed.

Decoding:

- constrained greedy decoding restricts each output codon to synonymous codons for the corresponding amino acid;
- decoder emits one codon per amino acid position;
- post-decoding validator is mandatory;
- invalid v3 candidates fall back to v2.

## Evaluation Metrics

Hard checks:

- amino acid identity = 1.0;
- no invalid codons;
- no internal stop codons;
- DNA length divisible by 3;
- validator passed.

Soft metrics:

- CAI;
- global GC;
- local GC windows;
- first 30/60/90 nt GC;
- forbidden motifs;
- homopolymers;
- repeats;
- codon usage distribution;
- feasibility-adjusted CAI/GC interpretation.

## Baselines

- v2 high-CAI pseudo-label baseline;
- v2 balanced baseline;
- current v3 checkpoint if available;
- native CDS reference;
- random synonymous baseline;
- constrained oracle from `factorforge.ml.feasibility` where tractable.

## Stop/Go Criteria

Stop before training if:

- pseudo-label JSONL lacks protein sequence metadata;
- synonym mask tests fail;
- dry-run model initialization fails;
- validator or fallback tests fail.

Stop during/after a small pilot if:

- decoded candidates fail amino acid preservation;
- v3 emits invalid codons or internal stops;
- special tokens appear in decoded CDS;
- v3 cannot match v2 safety metrics.

Proceed only if:

- validator pass rate is 100% on required benchmark candidates;
- v2 fallback is exercised and logged for invalid v3 candidates;
- feasibility analysis is used when strict CAI/GC targets fail.

## Ablation Plan

- v2 only;
- BART decoder without ESM2;
- ESM2 + BART decoder;
- random embeddings + BART decoder;
- v2 candidate set + rule-based reranker;
- v2 candidate set + ML reranker.
