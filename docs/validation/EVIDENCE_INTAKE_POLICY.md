# Evidence Intake Policy

Evidence may influence registry later, but no single evidence item directly changes stable defaults.

Required flow: Evidence Intake → Permission/visibility review → De-identification or hashing
→ Validation Evidence Package → Parameter Change Proposal → Maintainer Review
→ Registry Version Bump → Benchmark Re-run → Changelog

Negative and inconclusive evidence must be accepted. Evidence classes: positive / neutral / negative / inconclusive / not_comparable.
