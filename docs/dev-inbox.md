# Development Inbox

Use this page as a lightweight local inbox for notes that are not ready for a
GitHub issue. Keep entries short and move actionable items into issues when
they need tracking, discussion, or review.

## Capture Format

```text
Date:
Source:
Area:
Note:
Next action:
```

## Triage Rules

- Create a GitHub issue when the note affects users, releases, documentation, or
  CI reliability.
- Keep exploratory implementation ideas labeled as `idea`, `later`, and
  `claude-review`.
- Do not place secrets, private collaborator data, unpublished wet-lab details,
  or security vulnerability details in public issues.
- Report security concerns privately to `eijex.lab@gmail.com`.
