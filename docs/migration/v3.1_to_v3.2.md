# Migration Guide: v3.1 → v3.2

## User-facing behavior changes
- No breaking changes to CDS optimization output or CLI interface.
- Design Package JSON: `mfe_kcal_mol` is now `null` (not `0.0`) when MFE is not computed.
  New fields: `mfe_status` (`not_computed` | `computed` | `unavailable` | `failed`) and `mfe_used`.

## For library users
- New package: `factorforge.registry` — canonical parameter registry loader.
  `from factorforge.registry.registry_loader import load_registry, resolve_ref`
- `benchmarks/` directory added — formal benchmark runner and baselines.

## v3.2.0 is registry-audited, not fully registry-driven
Production optimization pipeline reads its defaults from code (not the registry at runtime).
The registry documents and drift-tests those values. Full registry-driven runtime is future scope.

## MFE null contract
Old (v3.1): `"mfe_kcal_mol": 0.0` when not computed.
New (v3.2): `"mfe_kcal_mol": null, "mfe_status": "not_computed", "mfe_used": false`.
