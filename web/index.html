                ⚠ This tool produces in-silico CDS design candidates and pre-synthesis review artifacts only. Review and wet-lab testing are required before relying on any design in experiments.

v3.4.0

                        2026-07-23
                    

                    
    
                        
  • Added: DNA/CDS and protein inputs now use distinct workflows; single-record FASTA is normalized and multi-FASTA is rejected.
    • 
                        
  • Fixed: 300 bp DNA is translated before synonymous redesign, preserving protein identity and nucleotide length instead of expanding to approximately 900 bp.
    • 
                        
  • Added: Configurable Required / Preferred / Ignored acceptance criteria, candidate-level PASS / CONDITIONAL PASS / FAIL decisions, QC matrix, reviewer disposition, and versioned provenance snapshots.
    • 
                    

                

                
                

                    

                    

                    

                        
v3.3.2

                        2026-07-07
                    

                    
    
                                                
  • Added: Added structured cai_authority metadata to CAI-bearing profile-engine and DP feasibility outputs, labeling whether CAI was evaluated against the golden-set reference, the generation reference, a fallback reference, or an unresolved reference id.
    • 
                        
  • Added: Added benchmark runner --out-dir and --force controls with an overwrite guard for existing result artifacts, plus a documented smoke/preflight/formal evidence-registration workflow.
    • 
                        
  • Fixed: Fixed profile-engine --reference-id CAI authority labeling so an explicit checksum-verified generation reference reports same_as_generation_reference instead of a fallback.
    • 
                        
  • Fixed: Fixed profile-engine --output FASTA headers to include engine=profile alongside profile, cai, gc, and score; profile FASTA headers intentionally omit DP-only objective metadata.
    • 
                    

                

                
                

                    

                    

                    

                        
v3.3.1

                        2026-07-03
                    

                    
    
                                                
  • Added: Added mfe_status_reason as a backward-compatible sub-reason for MFE provenance output, while keeping the existing mfe_status values unchanged.
    • 
                        
  • Added: Added a checksum-validated expert/research --reference-id CLI flag for selecting packaged codon-reference tables by manifest ID.
    • 
                        
  • Added: Added DP feasibility_best CAI-target exposure: CLI now accepts --cai-target, and the REST API accepts constraints.cai_target with default-preserving behavior plus explicit response echo for the requested DP target.
    • 
                        
  • Fixed: Fixed single-sequence profile-engine --output FASTA files so they include the same parseable metadata header used by batch/profile-comparison outputs, instead of writing only the raw CDS sequence.
    • 
                    

                

                
                

                    

                    

                    

                        
v3.3.0

                        2026-07-01
                    

                    
    
                        
  • Changed: Activated the NbeV1.1 high-confidence CDS-derived codon reference as the current N. benthamiana production software default, with the default GC policy band aligned to 40-47%.
    • 
                        
  • Added: Exposed public-safe reference-policy metadata and response fields for product version, active reference ID, GC reference band, and reference-policy version.
    • 
                        
  • Fixed: Removed selector-like public UI disclosure cards for legacy/research comparator codon-reference assets; comparator data remains packaged for provenance and controlled analysis only.
    •

This undoes a default-codon-reference promotion that shipped in v3.2.7, pending further validation.
